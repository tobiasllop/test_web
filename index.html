Muestreo para Definición de Clases

Para poder establecer los cortes óptimos de las clases sin saturar la capacidad de cómputo con la base completa del BCRA, aplicamos una técnica de muestreo sistemático.

Filtramos exclusivamente los registros cuyos CUIL/CUIT terminan en el dígito 0. Es fundamental aclarar que este muestreo se realizó solo para el proceso de definición de clases, garantizando una muestra representativa de la población.

Marco Teórico: El Método de Markov

Para el ranking de las entidades, implementamos el Método de Markov propuesto por Langville & Meyer en "Who’s #1?". Este enfoque trata al sistema financiero como un grafo dirigido donde el flujo de priorización de pagos determina la "fortaleza" de una entidad.

1. La Intuición: El Deudor como Votante

Utilizamos el modelo del "Hincha de Buen Tiempo" (Fair Weather Fan). Cada vez que un deudor tiene deudas con dos entidades (A y B) en la misma situación inicial, se genera un "partido". Si en el mes siguiente el deudor prioriza el pago a la entidad A sobre la B, la entidad B (perdedora) "emite un voto" de confianza hacia la entidad A (ganadora).

2. Etapas del Algoritmo

  • Matriz de Votos (V): Se contabilizan todas las derrotas relativas entre bancos.
  • Matriz Estocástica (S): Se normalizan los votos. Los bancos "invictos" se tratan como nodos sumideros, repartiendo su probabilidad equitativamente (1/n).
  • Matriz de Teleportación (S̅): Se aplica un Damping Factor (α = 0.85). Esto garantiza que el sistema sea irreducible y el ranking converja.
  • Vector Estacionario (π): El autovector principal determina el ranking, representando el "poder de atracción de pagos".
