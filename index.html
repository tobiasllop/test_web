Metodología Core

Marco Teórico: Modelado de Markov

1. El Deudor como Votante

Utilizamos el modelo del "Hincha de Buen Tiempo" (Fair Weather Fan). Cada vez que un deudor tiene deudas con dos entidades (A y B) en la misma situación inicial, se genera un duelo directo.

Si en el mes siguiente el deudor prioriza el pago a la entidad A sobre la B, la entidad B (perdedora) "emite un voto" de confianza hacia la entidad A (ganadora). Este flujo de capital revela la verdadera jerarquía ante restricciones presupuestarias.

2. Etapas del Algoritmo

  • Matriz de Votos (V): Se contabilizan todas las derrotas relativas entre bancos.
  • Matriz Estocástica (S): Normalización. Los bancos "invictos" se tratan como nodos sumideros, repartiendo probabilidad (1/n).
  • Damping Factor (S̅): Se aplica factor α = 0.85 para garantizar que el sistema sea irreducible.
  • Vector Estacionario (π): El autovector principal determina el ranking (poder de atracción de pagos).
